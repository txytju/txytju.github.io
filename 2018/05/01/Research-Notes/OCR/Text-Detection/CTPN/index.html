Structure of the model

Steps of the foward pass:

  1. From the raw image, we first get a feature map with FCN, which means that the size of feature map is not fixed.
  2. A dense sliding window in the feature map, on each position, we use a 3X3 window to get 3X3XC vector. So on each row, get W 3X3XC vectors.
  3. Use a Bi-LSTM to use context information, and convert 3X3XC vector to a 256 vector (128 each direction).
  4. For each position, using a FC layer, predict 2k coordinates, 2k scores and k side-refinements.

Key Ideas of this paper

  • This paper is some what related to Faster R-CNN, especially the RPN part.
    • Because there are only one target class to localization, so RPN of Faster R-CNN seems enough for plot bounding boxes and Fast R-CNN part is useless.
    • But the problem of RPN is the localization is not that accurate. As you can see in Faster R-CNN, a futher localization part is needed in Fast R-CNN to get more accurate localization.
    • So, if CTPN wants to build a text line localization based on RPN, the original RPN is not enough and some tricks are needed, as shown below.
  • Based on the characteristics of text lines, some tricks are as follows:
    • Text line can be long or short and the length of text line can vary a lot. So narrow anchors are applied to detect some text, rather than the whole text line!
    • A Bi-LSTM is applied, ensuring the context information in deciding whether an anchor is text or not.
  • Why this method is better in localization?
    • Compared to the RPN or Faster R-CNN system, our fine-scale detection provides more detailed supervised information that naturally leads to a more accurate detection.

Some Information about OCR

  • Poor performance of RPN on text

    • Text detection is defined in word or text line level, so that it may be easy to make an incorrect detection by defining it as a single object, e.g., detecting part of a word. Therefore, directly predicting the location of a text line or word may be difficult or unreliable, making it hard to get a satisfied accuracy, as shown in the image below.
